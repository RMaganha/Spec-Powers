## 0.8.5 — 2026-07-20 (CI com artefatos de teste)
- feat(ci): **CI com artefatos de teste** (item 12, depende do 9) — novo `.gitlab-ci.yml` na raiz: um job `test` que instala `pytest pytest-cov PyYAML`, roda a suíte e publica os artefatos que o **GitLab renderiza nativamente**: **JUnit XML** (`--junitxml` → resultados + **tendência de duração** ao longo dos runs, via o `time` de cada teste) e **Cobertura XML** (`--cov --cov-report=xml` → cobertura no MR/diff). Os flags de artefato ficam **no comando do job, não no `addopts`** do pytest — o `pytest -q` local segue limpo e sem exigir `pytest-cov`. As **saídas de run** (`report.xml`, `coverage.xml`, `htmlcov/`, `.coverage`) entram no `.gitignore`: run output **nunca** é commitado (o anti-padrão que o item 12 evita). **Preparado, NÃO ativado** (placeholder, como o item 9): `git remote` está vazio, então apontar/rodar num host é passo manual do owner quando o host git interno existir — nada aponta pra host inventado. Só config + gitignore + wiring: `tests/test_ci.py` (5 ACs) trava estrutura, flags, gitignore e o placeholder. Comando exercitado localmente com `pytest-cov`: gerou JUnit + Cobertura válidos, `git check-ignore` confirmou run output ignorado. Spec: `docs/superpowers/specs/2026-07-20-ci-artefatos-teste-design.md`.
