# Decisões — mss-spec

- 2026-07-16 — segredo de conexão via **variável de ambiente** (`.env` dev → Azure App Settings prod) em vez de **Fernet-no-código** como padrão — porque a maioria é web app Azure e o Fernet com a chave no repo é só ofuscação; Fernet fica como opção de continuidade.
- 2026-07-16 — **robustez do `${CLAUDE_PLUGIN_ROOT}`** virou **checagem no doctor + guard no kickoff** em vez de comando próprio — porque um comando não roda se o plugin não resolve (circular); o CLAUDE.md/doctor está sempre presente e avisa.
- 2026-07-16 — `doctor` é **manual + automático** (regra no CLAUDE.md, na 1ª tarefa) em vez de só comando — porque erro de ambiente tem que aparecer cedo, não no build.
- 2026-07-16 — `upgrade` **mescla** `CLAUDE.md`/`AMBIENTE.md` (conflito pergunta) e **não mexe em código** sozinho — porque não pode destruir a customização do projeto nem alterar `get_connection.py` às cegas.
- 2026-07-16 — `to-dolist` fica **fora do git** (`.gitignore` ancorado `/to-dolist.md`) em vez de versionado — porque assim aparece em qualquer branch (arquivo não-rastreado não é tocado no checkout) e é captura pessoal.
- 2026-07-16 — superpowers seria declarado como **dependência** (auto-install), mas foi **revertido**: a dependência **cross-marketplace** quebra o carregamento quando o plugin roda via **skills-dir/symlink** (`~/.claude/skills/mss-spec` → repo) — a allowlist só vale em load por marketplace, que não é lido nesse caminho, então o Code recusa o plugin e somem todos os comandos. Fica pro passo do **marketplace git** (item 9); por ora, superpowers segue habilitado via `settings.json`.
- 2026-07-16 — `anotar decisões` = **um `docs/decisoes.md`** (só transversais) em vez de subsistema `docs/adrs/` — porque decisão por-assunto já está no Histórico da spec; um arquivo scannável basta.
- 2026-07-16 — backlog do projeto (necessidades levantadas no `kickoff`) vai no **`docs/superpowers/INDEX.md`** (semeado pelo `kickoff` no fecho) em vez do `to-dolist` — porque o INDEX é versionado/compartilhado (o backlog é do projeto) e o `to-dolist` é pessoal/fora do git; e reusa o índice que já existe (sem comando novo).
- 2026-07-16 — padrão de **log**: `logging` da stdlib pro **stdout** (o Azure captura em prod) **+** arquivo rotativo em `logs/` só em **dev** — porque arquivo dentro do container Azure é efêmero (some no restart/deploy); liga/desliga no `.env` (`LOG_ATIVO`). Spec em `docs/superpowers/specs/2026-07-16-log-design.md`.
- 2026-07-17 — **`compliance` audita, `upgrade` conserta**: a checagem de aderência às convenções (`/mss-spec:compliance`) é **checklist fixo, só reporta**, em vez de detectar drift comparando com os templates atuais — porque quem faz diff/merge com o molde é o `upgrade`; papéis separados evitam dois comandos mexendo na mesma coisa. compliance confere **presença/wiring** (estrutura, docs, memória, gitignore); a auditoria AppSec profunda fica no `seguranca`. Trio de auditoria: **doctor** (ambiente) · **seguranca** (AppSec) · **compliance** (convenção).
- 2026-07-20 — **CI do kit = GitLab CI** (`.gitlab-ci.yml`) em vez de GitHub Actions/Azure Pipelines — porque o GitLab renderiza **nativamente** os três artefatos do item 12 (JUnit XML → resultados + tendência de duração; Cobertura XML → cobertura no MR) sem plugin, e casa com git self-hosted interno. Os flags de artefato (`--junitxml`/`--cov`) ficam **no comando do job, não no `addopts`** do pytest — pra o `pytest -q` local seguir limpo e sem exigir `pytest-cov`. Preparado, **não ativado** (placeholder, como o item 9): sem host, ativar é passo do owner. Spec: `docs/superpowers/specs/2026-07-20-ci-artefatos-teste-design.md`.
- 2026-07-20 — **mapa de contexto = fonte federada** (cada repo declara suas Conexões no próprio `docs/superpowers/MAPA.md`) em vez de **registro central** — porque fato de integração perto do código não mente e escala por projeto; o **mapa mental** (F2) só **deriva** do repo (lê a seção Conexões do `MAPA.md` local + escaneia o código), nunca é a fonte. Distinto dos **precedentes** (`precedentes-msig` = padrão a copiar): Conexões = integração de runtime "o que vai pra onde". Spec: `docs/superpowers/specs/2026-07-20-mapa-de-contexto-design.md`.
- 2026-07-20 — **mapa neural = mapa MENTAL do projeto** (projeto no centro + 4 dimensões extraídas do repo: arquitetura, APIs & integrações, memórias, conexões), HTML radial **full-screen** expansível/arrastável **self-contained** (SVG + JS vanilla, zero CDN — proxy MSIG) — em vez de só um grafo de conexões inter-projeto. Isso **reabriu** `knowledge-graph` (dimensão de memórias) e `dependency-graph` (arquitetura interna), antes "não fazer" no INDEX — reversão consciente do owner ao ver o protótipo pobre. Extração **estática/heurística** (rota por decorator, integração por `import`, camada por pasta; ignora `tests/`), não análise semântica. Spec: `docs/superpowers/specs/2026-07-20-mapa-de-contexto-design.md`.
- 2026-07-17 — **log é opt-in por-arquivo, não global**: o `kickoff` monta só a **infra** (`config/logging.py` + `.env`/`.gitignore`); a **instrumentação** (`logger = getLogger(__name__)`) é escolhida arquivo-a-arquivo — em vez de instrumentar tudo — porque script/util simples não precisa e ruído polui. **Regra transversal:** todo comando que **gera arquivos** lista os alvos (resumo de 1 linha cada) e pergunta "logs em todos ou em quais?" antes de escrever (banco/API/regras sim; simples não). Ícone por nível e arquivo rotativo são **só em dev** (Azure = stdout limpo, detectado por `WEBSITE_SITE_NAME`).
- 2026-07-21 — **memória em 3 camadas + captura consolidada no `/mss-spec:memory capturar`** (fatos · decisões incl. negativas · **diário de sessão** datado/indexado `memory/sessions/` + `DIARIO.md`), com gatilho **determinístico no fecho** + **hook opt-in só como rede** — em vez de integrar o **claude-mem** (serviço Bun + SQLite + Chroma vetorial que auto-captura o firehose) ou criar comando novo — porque casa com os pilares (memória **curada**, **não-serviço**, "texto consultável, não busca vetorial — knowledge-graph fora de escopo") e não incha o plugin (o `MAPA.md` segue dono do `/mss-spec:mapa`; o hook nunca é a fonte da verdade). Spec: `docs/superpowers/specs/2026-07-21-captura-de-memoria-design.md`.
